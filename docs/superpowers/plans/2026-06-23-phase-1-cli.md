## Task 1: `ConfigStore` (persistencia de `~/.murmur/config.json`)

**Files:**

- Create: `packages/cli/src/config.ts`
- Test: `packages/cli/src/config.test.ts`

Contrato:

```ts
export type Theme = 'system' | 'dark' | 'light';

export interface MurmurConfig {
  openaiApiKey?: string;
  hotkey: string;
  model: string;
  voice: string;
  theme: Theme;
}

export const DEFAULT_CONFIG: Omit<MurmurConfig, 'openaiApiKey'> = {
  hotkey: 'CommandOrControl+Shift+Space',
  model: 'gpt-realtime',
  voice: 'alloy',
  theme: 'system',
};

export class ConfigStore {
  constructor(baseDir?: string); // default: process.env.MURMUR_HOME ?? join(homedir(), '.murmur')
  baseDir(): string;
  path(): string; // <base>/config.json
  dataPath(file: string): string; // <base>/<file>  (p. ej. memory.db)
  load(): MurmurConfig; // defaults si no existe; ConfigError si JSON inválido
  save(patch: Partial<MurmurConfig>): MurmurConfig; // merge + escribe 0600 + mkdir -p
  setOpenAiKey(key: string): MurmurConfig;
}
```

Pasos:

- [ ] Test (falla): roundtrip save/load en `mkdtemp`; `load()` sin archivo devuelve defaults;
      archivo escrito tiene modo `0600` (comprobar con `fs.statSync(...).mode & 0o777 === 0o600`);
      JSON malformado (`writeFileSync` con `"{"`) → `load()` lanza `ConfigError`.
- [ ] Implementar `config.ts`: usa `node:fs`, `node:os`, `node:path`. `save` hace
      `mkdirSync(base, { recursive: true })`, `writeFileSync(path, JSON.stringify(merged, null, 2), { mode: 0o600 })`, y además `chmodSync(path, 0o600)` (por si el archivo ya existía con otros perms). Merge sobre el `load()` actual + defaults.
- [ ] Verde: `pnpm --filter murmur test`.
- [ ] Commit: `feat(cli): ConfigStore con persistencia en ~/.murmur/config.json`.
